## 1. 배포 아키텍처 명확화

### 1.1 EC2 배포 구성

**배경**: 평균 100ms 이하의 응답 시간 목표와 중규모 동시 사용자(50-200명)를 고려할 때, EC2 배포 구성에 대한 명확화가 필요합니다.

**질문**: EC2 배포 시 어떤 구성을 원하시나요?

**옵션**:
A) 단일 EC2 인스턴스 (간단한 구성, MVP 테스트용)
B) 다중 EC2 + Application Load Balancer (고가용성, 부하 분산)
C) Auto Scaling Group + ALB (자동 확장, 트래픽 대응)
D) 단일 EC2로 시작하되, 향후 확장 가능한 구조로 설계
E) Other (다른 구성을 선호하시면 아래에 설명해주세요)

[Answer]:A 


### 1.2 응답 시간 목표 재확인

**배경**: 평균 100ms 이하는 매우 도전적인 목표입니다. 선택하신 배포 구성과의 정합성을 확인하고자 합니다.

**옵션**:

C) 조정 - 평균 500ms 이하 (일반적 수준, MVP에 적합)

## 2. 이미지 관리 명확화

### 2.1 메뉴 이미지 초기 데이터 입력

**배경**: 메뉴 관리 기능은 제외하되, 이미지 업로드 기능은 포함하셨습니다. 초기 메뉴 데이터(이미지 포함)를 어떻게 입력할지 명확화가 필요합니다.

**질문**: 초기 메뉴 데이터(이미지 포함)는 어떻게 입력하시겠습니까?

**옵션**:
A) 데이터베이스 직접 입력 + S3에 수동 업로드 (개발자가 직접 처리)
B) 별도 관리자 도구 개발 (간단한 메뉴 등록 화면만)
C) API 직접 호출 (Postman 등으로 메뉴 등록 API 호출)
D) 메뉴 관리 기능 포함으로 변경 (기본 CRUD 구현)
E) Other (다른 방식을 선호하시면 아래에 설명해주세요)

[Answer]:D 


### 2.2 이미지 업로드 기능 범위

D) 메뉴 관리 기능 없이 이미지 업로드 API만 구현 (API로만 사용)
